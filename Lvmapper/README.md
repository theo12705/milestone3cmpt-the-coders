How the official artifact was discovered:
It was undiscovered, below I have listed the methods I have used to try to locate the offical version of the tool. 

First line of search was using key word searches like "LVMapper Github", "LVMapper Clone Detector", "LVMapper Clone Detection". None 
of these searches had brought up the original repository that belongs to LVmapper. 


So decided to pivot my search to look through the articles that were appearing in the above searches that go over the results of LVMapper, hoping that there 
is a reference to a rpository or some sort of code base. I went through the same article in two different locations in case there was a reference section I missed 
that was hidden due to the website format. The links are below. Still couldnt find the codebase this way. 
https://clones.usask.ca/pubfiles/articles/MingWuLVMapperIEEEAccess2020.pdf
https://arxiv.org/abs/1909.04238


So after changing up search parameters, adding in keywords like 'Experiment' 'Comparisons' to see if I could find someone that used this repo
as either an enhancement for a tool or comparing against other clone detectors I had found one repository that did this, but didnt have the source code 
available. When listing the tools for comparison it says thatLVMapper is not avaiable to the public and needs to be requested by authors. The repo I found this 
information in is linked below:
https://github.com/kusumotolab/NIL/blob/master/EXPERIMENT.md

Afterwards I decided to try to search for each authors github accounts but the only reliable one I could find is professor Roy's which was empty with no 
content in it. 

The final step in the search is to use Google Gemini's search features to find either official profiles for each author or a repository listing with LVMapper in it but there were no results with this method either. I can only conclude after this point that it is truly not avaiable to the public and must be requested from the authors.
