## How the official artifact was discovered:
**There was no tangible software artifact discovered.**

The official paper for this tool was found [here](https://ieeexplore.ieee.org/document/8665855) by searching Google for `tcore "a novel approach for detecting type-iv clones in test code"`. However, there was no link to a repository to be found in the references nor the paper itself. The paper says that this is an "open-source tool" and cites [another reference](https://dl.acm.org/doi/abs/10.1145/3242163.3242168) for this fact, but that also contains no reference to a repository. Searching for ["t-core"](https://www.google.com/search?client=firefox-b-d&q=t-core) and ["tcore"](https://www.google.com/search?q=tcore) reveal only [irrelevant](https://github.com/PRBonn/TCoRe) [repositories](https://github.com/yafeng19/T-CORE).

I then looked into the authors of this paper, Brent van Bladel and Serge Demeyer. [Demeyer's Github account](https://github.com/sergedemeyer) shows [no results for T-CORE](https://github.com/search?q=owner%3Asergedemeyer%20t-core&type=code), [nor TCore](https://github.com/search?q=owner%3Asergedemeyer+tcore&type=code). [Bladel's account](https://github.com/brentvanbladel) appears to have been deleted entirely. Searching Github itself for any possible forks [does](https://github.com/search?q=t-core+serge+demeyer&type=code) [not](https://github.com/search?q=t-core+brent+van+bladel&type=code) [give](https://github.com/search?q=tcore+brent+van+bladel&type=code) [any](https://github.com/search?q=tcore+serge+demeyer&type=code) results.

The authors appear to have written the paper under the University of Antwerp in Belgium, so I tried [looking into that](https://www.google.com/search?q=university+of+antwerp+%22t-core%22), but did not show any relevant results. A ["tools" page](https://repository.uantwerpen.be/docman/irua/0fdb14/193139.pdf) from the University of Antwerp returns a 404, and [another paper on the same topic](https://www.uantwerpen.be/en/research-groups/ansymo/tools/) from the same authors also does not give any references to any sort of link to any software artifacts.

Searching [Google Scholar](https://scholar.google.ca/scholar?hl=en&as_sdt=0%2C5&q=t-core+serge+demeyer&btnG=&oq=t-core+serge+demeyer) only redirected me to the papers I found before. In a final act of desperation, I even asked Google Gemini to find any information, but [that attempt](https://gemini.google.com/share/768be8692638) was fruitless.
